# Definition of Done (DoD)

---
## Sprint 1 считается завершённым, если:

### Функциональность
- [ ] Реализованы CRUD операции для задач
- [ ] Корректные HTTP-коды (200, 201, 204, 400, 404)
- [ ] Валидация входных данных работает
- [ ] Swagger доступен по /swagger

### Архитектура
- [ ] Проект разделён по слоям
- [ ] Контроллеры не используют DbContext напрямую
- [ ] Используется Dependency Injection

### База данных
- [ ] Настроены миграции EF Core
- [ ] База создаётся через Migrate(), а не EnsureCreated()
- [ ] Таблицы Users, Tasks, Tags, TaskTags созданы корректно

### Качество
- [ ] Написаны минимум 2–3 unit-теста
- [ ] Тесты проходят
- [ ] README содержит инструкцию запуска

---
## Sprint 2 считается завершённым, если:

### Фильтрация
- [ ] Работает фильтр по статусу
- [ ] Работает фильтр по исполнителю
- [ ] Работает фильтр по дедлайну
- [ ] Работает фильтр по тегам
- [ ] Фильтрация выполняется на уровне БД (LINQ to Entities)

### Бизнес-логика
- [ ] Статус Overdue вычисляется runtime
- [ ] CompletedAt автоматически заполняется при Status = Done
- [ ] CompletedAt сбрасывается при возврате из Done

### Тестирование
- [ ] Unit-тесты на IsOverdue
- [ ] Unit-тест на автозаполнение CompletedAt
- [ ] Нет флейков из-за DateTime

### Отчёты
- [ ] Реализован отчёт по статусам
- [ ] Реализован отчёт по просроченным задачам

---
## Sprint 3 считается завершённым, если:

### UI
- [ ] Есть страница списка задач
- [ ] Есть форма создания задачи
- [ ] Есть форма редактирования задачи
- [ ] Работают фильтры через UI
- [ ] UI работает без использования Swagger

### Отчёты
- [ ] Реализован отчёт по среднему времени закрытия
- [ ] Отчёты отображаются через UI

### Качество
- [ ] Минимум 5 unit-тестов в проекте
- [ ] Все тесты проходят
- [ ] Нет критических багов
- [ ] Проект запускается локально одной командой

---
## Проект считается успешно завершённым, если:
- [ ] Реализованы все CRUD операции
- [ ] Работают фильтры
- [ ] Реализованы минимум 2 из 3 отчётов
- [ ] UI полностью функционален
- [ ] Есть unit-тесты
- [ ] Swagger корректно документирует API
- [ ] Код хранится в Git
- [ ] Используется Pull Request (после стартового коммита)
